## 2️⃣ 加新案件（最重要！）

### 步骤一：用 VS Code 打开 `cases.js`

1. 打开 VS Code
2. 拖动 `cases.js` 文件到 VS Code 窗口里（或 File → Open）

### 步骤二：照着已有案件复制粘贴

文件里每个案件都长这样（一个 `{ ... }` 块）：

```javascript
{
  province: "浙江省",
  city: "杭州",
  title: "梁某诉某科技公司生成式人工智能\"幻觉\"侵权案",
  caseNumber: "",
  year: 2025,
  tech: ["AI幻觉"],
  claim: ["人格权"],
  note: "",
  status: "decided",
  teamArticle: true,
  url: "https://mp.weixin.qq.com/s/..."
},
```

**加新案件的步骤**：

1. 在文件里找一个最像你要加的新案件的现有案件
2. 选中它整个 `{ ... },`（**记得连同最后的逗号**），按 `Ctrl+C`（Mac: `Cmd+C`）
3. 在它后面新一行，按 `Ctrl+V` 粘贴一份
4. 把粘贴的那份里的内容改成新案件的信息
5. 按 `Ctrl+S`（Mac: `Cmd+S`）保存
6. 双击 `index.html` 刷新页面（如果已经打开了，按 F5 或 Cmd+R 刷新）

### 字段填写参考

| 字段 | 填什么 | 注意 |
|---|---|---|
| `province` | 省份/直辖市的全名 | **必须用全名**，比如 `"北京市"` 不是 `"北京"`；`"广西壮族自治区"` 不是 `"广西"` |
| `city` | 城市或法院名 | 比如 `"杭州"`、`"北京互联网法院"`、`"浦东新区"` |
| `title` | 案件名 | 中文双引号 `"" `要改成英文双引号 `\"` 转义，或者直接用英文双引号 |
| `caseNumber` | 案号 | 没有就填 `""`（两个英文引号挨着） |
| `year` | 年份 | 数字，**不要加引号**，如 `2026` |
| `tech` | 技术类型 | 数组，可以多个：`["AI文生图", "AI换脸"]` |
| `claim` | 法律权益类型 | 同上，数组 |
| `note` | 裁判要旨/备注 | 比如 `"AI生成图片不构成作品"`，没有就 `""` |
| `status` | 状态 | 只能填 `"decided"`（已审结）或 `"pending"`（审理中） |
| `teamArticle` | 是否团队文章 | 只能填 `true` 或 `false`（**不要加引号**） |
| `url` | 链接 | 必须以 `"https://` 或 `"http://` 开头 |

### ⚠️ 常见错误（90% 的问题都是这些）

1. **忘了逗号**：每个 `{ ... }` 后面要有个 `,`（最后一个案件不需要）
2. **中文标点**：所有 `"` 必须是英文双引号，不是中文 `"`；`,` 必须是英文逗号，不是中文 `，`
3. **缺少字段**：所有字段都要有，没值就填空字符串 `""` 或空数组 `[]`
4. **case 拼写**：`true/false` 都是小写

**如果出错了**，刷新网页会变成白屏。这时候打开 VS Code 找到你刚加的那段，对照模板检查这几个常见错误。**不会损坏其他东西**，最坏情况删掉你新加的那段，回到原始状态。

### 💡 备份小习惯

每次大改前，把整个文件夹复制一份当备份。免费、安心。

## 3️⃣ 上线到 GitHub Pages（让宋律师能在线访问）

这一步**只需要做一次**，之后每次改 `cases.js` 上传新版本即可。预估时间 **30 分钟**。

### 步骤一：创建 GitHub 仓库

1. 登录 GitHub.com（你已经有账号了 ✓）
2. 右上角点 `+` → `New repository`
3. 填仓库名（比如 `china-ai-cases` 或 `kwm-ai-map`，建议用英文/拼音）
4. **Public**（公开）这个选项要选上，**否则 GitHub Pages 免费版用不了**
   - ⚠️ 注意：这意味着任何人能看到你的代码。但 case 数据本来就是公开案例，无所谓。
   - 如果想要私有，需要付费 GitHub Pro，那就不用这个方案
5. ⚠️ 不要勾选 "Add a README file"
6. 点 `Create repository`

### 步骤二：上传文件

1. 在新创建的仓库页面，会看到一行小字 `uploading an existing file`，点它
2. 把整个 `china-cases` 文件夹里的 **6 个文件**（不是文件夹本身，是里面的内容）拖到上传区域
3. 等所有文件显示绿色 ✓
4. 页面下方有个绿色按钮 `Commit changes`，点它

### 步骤三：打开 Pages 功能

1. 仓库顶部菜单 → `Settings`（齿轮图标）
2. 左侧菜单 → `Pages`
3. `Source` 这一栏选 `Deploy from a branch`
4. `Branch` 这一栏选 `main`（或 `master`），文件夹保持 `/ (root)`
5. 点 `Save`
6. 等 **1-3 分钟**，刷新这个 Pages 页面，最上方会出现一个绿色框写着 `Your site is live at https://你的用户名.github.io/仓库名/`

**这个 URL 就是你的网站链接！** 发给宋律师就行。

### 步骤四：以后更新（重要）

下次你改了 `cases.js`，怎么更新线上版本？

1. 进入 GitHub 仓库
2. 点 `cases.js` 文件
3. 右上角铅笔图标 ✏️（"Edit this file"）
4. 直接在浏览器里改（或者删除内容后从本地复制粘贴新内容）
5. 滚到底部，点绿色 `Commit changes` 按钮
6. 等 **1-3 分钟**，刷新网站，新案件就出现啦

> 💡 **更简单的方法**（适合频繁更新）：去 GitHub Desktop 桌面应用（免费下载），用图形界面 sync 本地和线上仓库，不用打字。

## 🆘 出问题怎么办

| 现象 | 大概率原因 | 解决 |
|---|---|---|
| 双击 index.html 是白屏 | `cases.js` 里有语法错误 | 把你最近改的那段删掉，恢复保存 |
| 字体看起来不对 | 没联网/字体加载失败 | 不影响功能，会自动回退到系统字体 |
| GitHub Pages 显示 404 | Pages 还在部署中 | 等 5 分钟再刷新 |
| 地图不显示 | 浏览器太老 | 用 Chrome / Edge / Safari 最新版 |
